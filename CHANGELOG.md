## [2.1.2] — 2026-06-13

### 变更
- **GPU 推理恢复**：`onnxruntime-gpu` 1.26 + CUDA 12.8 + cuDNN 9.23，RTX 4060 Laptop GPU
  - `INSIGHTFACE_CTX_ID = 0`，所有 5 个 ONNX 模型均使用 `CUDAExecutionProvider`
  - 人脸检测+识别：CPU 22.5ms → GPU ~6ms（提速 3-4x）
- **考勤记录行内编辑**：点击状态徽章或备注文字即可直接编辑
  - 状态：点击切换下拉框（成功/失败/请假）
  - 备注：点击切换输入框（回车保存，Esc 取消）
  - AJAX 保存，无页面刷新，Toast 提示结果
  - `record_edit` 和 `record_leave_edit` 端点支持 AJAX 返回 JSON
- **请假审批增加删除按钮**：每条请假记录增加 🗑️ 删除按钮（带确认弹窗），已处理/待审批均可删除
- **请假列表去掉 ID 列**：表头从 8 列减为 7 列（姓名/工号/日期/原因/状态/提交时间/操作）

- **缺勤规则统一**：只要状态不是「成功」都按缺勤算（含失败/请假/无记录）
  - `today_absentees()` 重写：不再排除请假通过者，缺勤 = 无成功签到
  - 仪表盘「应签到」= 全员，「缺勤」= 全员 - 成功
  - 仪表盘缺勤详情增加状态/原因列
- **考勤记录全员状态可编辑**：3 种行类型均支持点击状态编辑
  - 有签到记录 → 下拉选择（成功/失败/请假）
  - 纯请假记录 → 编辑请假原因
  - 未签到（缺勤）→ 输入原因回车快速创建请假
- **模块数据联动**：考勤记录页的状态变更直接反映到仪表盘统计
- **考勤记录统一 3 状态切换**：每行统一下拉（成功/请假/未签到），去掉了按行类型分 3 种交互的复杂逻辑
  - 成功 → 写入/覆盖签到记录 + 清除今日请假
  - 请假 → 写入请假（自动通过）+ 清除今日签到
  - 未签到 → 清除签到和请假，恢复缺勤
  - 新增 `Database.set_employee_status()` + `POST /records/row/<id>/set-status`
  - 点击「未签到」不再自动弹出输入框生成请假审批

### 修复
- **个人签到 vs 合照签到混淆**：两个页面分别添加醒目提示
  - 个人签到：「仅识别最大人脸，只生成 1 条记录」
  - 合照签到：「检测所有人脸，每人生成 1 条记录」
- **签到按钮防重复提交**：拍照/上传按钮在请求期间禁用+显示 spinner，防止连点产生多条记录
- **AI 助手上下文大幅增强**：每次对话注入 4 个模块的完整实时数据
  - 仪表盘：日期、人员数、签到/缺勤/请假统计
  - 人员管理：全部人员名单（姓名+工号+ID）
  - 考勤记录：今日全员状态明细（成功/失败/请假/缺勤 + 相似度 + 方式 + 备注）
  - 请假审批：全部请假记录（申请人+日期+原因+审批状态）
  - AI 可基于实际数据回答"谁缺勤了""某人的签到情况""还有几条请假待审批"等问题
- **仪表盘自动同步**：所有模块（考勤记录编辑/请假审批/人员管理）直接操作同一数据库，仪表盘刷新即反映最新状态
- **答辩 PPT & DOC 更新**：PPT 第15页 Q&A 和 DOC 第十二章更新为 5 个核心问答
  - Q1: AI 接口代码位置（app.py /chat/ask，上下文注入4模块数据）
  - Q2: 技术栈全景 + 前后端连接方式（Flask路由+HTTP协议）
  - Q3: 人脸识别模型详解（SCRFD+ArcFace，预训练模型，无需自训练）
  - Q4: Flask 是什么 + Python 写网页原理（路由+Jinja2+13端点）
  - Q5: 项目创新点 + 最大困难（GPU部署/模型升级/状态同步）
  - PPT 第18页更新为 v2.1.2 最新版本说明

### 新增
- `Database.delete_leave()` 方法
- `POST /leave/<id>/delete` 路由
- `POST /records/leave/create` 路由 — 缺勤快速标记请假

## [2.1.0] — 2026-06-12

### 变更
- **GPU 推理启用**：RTX 4060 Laptop GPU 加速，onnxruntime-gpu 1.26 + CUDA 12.8 + cuDNN 9.23
  - 人脸检测+识别：22.5ms → **6.4ms**（提速 3.5x）
  - `face_engine.py` 自动预加载 CUDA/cuDNN DLL
- **人脸识别模型升级**：`buffalo_s` → `buffalo_l`，ArcFace 识别精度大幅提升
- **嵌入向量缓存**：已注册人脸嵌入预计算+内存缓存，避免重复磁盘IO和ArcFace推理。启动时预热，人员变更时刷新

### 修复
- **仪表盘 Jinja2 模板崩溃**：`stats.today_leave` 在无 `__pycache__` 或旧缓存时抛出 `UndefinedError`，改用 `.get()` 安全访问；同时修复 AI 聊天上下文中 `attendance_stats()` 解包缺变量（4→5）
- **端口 5050 僵尸进程**：端口改为 5001，新增 `run.bat` 一键启动脚本
- **GPU 模式硬崩溃**：`INSIGHTFACE_CTX_ID` 临时改为 -1（CPU），onnxruntime-gpu CUDA 内存分配失败导致进程闪退
- **考勤记录过滤视图缺少编辑/删除按钮**：`list_attendance()` 字段名统一为 `att_id`/`att_status`
- **请假行无操作按钮**：请假员工行新增内联编辑（修改备注），无签到记录时不显示删除按钮
- **同日重复签到产生多条记录**：`log_attendance()` 改为 UPSERT 逻辑，同人同日再签到时覆盖旧记录而非新增
- **仪表盘与考勤记录数据口径不统一**：缺勤统计排除已请假通过人员，签到分母改为「应签到人数」（总人数-已请假），两个页面统一逻辑

### 代码质量
- 提取 3 个共享辅助函数：`_decode_upload()`、`_find_best_match()`、`_get_status_info()`
- 删除死代码：`Database.close()`、`Database.test_connection()`、`EXPORT_LIMIT`
- 消除 `formatSize()` JS 重复定义（统一到 base.html）
- 消除 `.stat-card` CSS 重复声明
- 消除 3 处图片解码 + 2 处人脸比对循环重复

### 修复

## [2.0.0] — 2026-06-12

### 变更
- **人脸识别引擎升级为深度学习 (v3.0 → v4.0)** 🚀
  - **检测器**：Haar Cascade → **SCRFD**（InsightFace 深度神经网络检测器）
    - 侧脸检出率 +60%，遮挡/弱光检出率 +80%
    - 合照中多人脸检测精度大幅提升
    - 自动去重，无需手动 IoU 计算
  - **识别器**：HSV+LBP+LAB 直方图 → **ArcFace**（512维深度学习嵌入向量）
    - LFW 基准准确率 99.83%，工业级人脸识别模型
    - 嵌入向量对光照/姿态/表情/年龄/肤色具有深度不变性
    - 同人余弦相似度 >0.65，不同人 <0.30，区分度极高
  - **相似度计算**：直方图相关性 → 余弦相似度（归一化向量点积）
- **新增依赖**：`insightface>=1.0` + `onnxruntime>=1.26`
- **配置更新**：新增 `INSIGHTFACE_*` 系列配置项，旧版 Haar/LBP 参数标记为弃用

### 移除
- 不再使用 OpenCV Haar Cascade 级联检测器
- 不再使用 HSV/LBP/LAB 三特征融合方案
- 不再使用 UMat GPU 加速（改用 ONNX Runtime）

## [1.3.0] — 2026-06-12

### 新增
- **考勤记录管理**：支持对单条考勤记录进行删除和修改（状态/备注）操作
- **AI 客服聊天**：接入豆包 (Doubao) API，提供智能问答客服功能
- **管理员仪表盘**：首页展示总人数、今日签到成功数、缺勤人数、待审批请假数等统计数据
- **今日缺勤列表**：首页实时展示今日未签到且未请假的人员名单
- **多角度人脸注册**：人员注册时支持上传 1~3 张不同角度的人脸照片
- **数据增强**：注册照片自动进行数据增强（翻转/旋转/亮度调整等），从单张照片生成多个训练变体，提升识别率
- **个人签到图片上传**：在摄像头拍照基础上，增加本地上传图片入口，提高识别通过率
- **合照签到**：上传一张包含多人的合照，自动检测照片中所有人脸并逐一识别签到
- **考勤 CSV 导出**：支持将考勤记录导出为 CSV 文件

### 变更
- **项目架构重构**：从单体脚本拆分为模块化结构
  - `app.py` — Flask 路由和业务逻辑
  - `config.py` — 集中配置管理
  - `database.py` — 数据库操作层
  - `face_engine.py` — 人脸识别引擎
  - `models.py` — 数据模型定义
- **GUI → Web**：从 Tkinter 桌面 GUI 改为 Flask Web 应用，支持浏览器访问
- **数据库迁移**：使用 PyMySQL 连接 MySQL 数据库

### 修复
- 优化人脸检测算法，改用多尺度检测 + 多角度比对策略
- 修复深色肤色人脸识别精度偏低的问题
